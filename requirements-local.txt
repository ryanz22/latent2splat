# Local 3DGS-decoder training env for an RTX 5090 (Blackwell, SM 12.0 / sm_120).
#
# CRITICAL: the 5090 needs CUDA 12.8+ and PyTorch >= 2.7.0 — 2.7.0 was the first
# stable release with native sm_120 prebuilt wheels. torch 2.10.0+cu130 with
# TORCH_CUDA_ARCH_LIST="12.0" (what docker/Dockerfile.decoder uses) also works;
# pick ONE CUDA line and keep torch + gsplat on the same toolkit.
#
# Install order matters: install torch FIRST (it pins the CUDA runtime gsplat
# builds against), then build gsplat, then the rest.

# --- Step 1: PyTorch with CUDA 12.8 (sm_120). Run as its own command: ---
#   pip install torch==2.7.0 torchvision==0.22.0 --index-url https://download.pytorch.org/whl/cu128
# (Or match production: torch==2.10.0 torchvision --index-url .../whl/cu130 — then
#  use CUDA 13.0 toolkit for the gsplat build below.)

# --- Step 2: build gsplat against the installed torch (needs nvcc + ninja): ---
#   TORCH_CUDA_ARCH_LIST="12.0" pip install --no-build-isolation gsplat==1.4.0
# gsplat JIT/AOT-compiles CUDA extensions; the ARCH_LIST must include 12.0 so it
# emits sm_120 kernels. If the prebuilt wheel lacks sm_120, build from source:
#   TORCH_CUDA_ARCH_LIST="12.0" pip install --no-build-isolation \
#     git+https://github.com/nerfstudio-project/gsplat.git

# --- Step 3: the rest (CUDA-independent) ---
numpy<2
Pillow
pytest

# Notes:
# - System deps for the gsplat build: a CUDA toolkit matching the torch wheel
#   (nvcc on PATH), ninja, and a C++ compiler. On Ubuntu: apt install ninja-build.
# - Verify sm_120 after install:
#     python -c "import torch; print(torch.__version__, torch.cuda.get_device_capability())"
#   should print (12, 0) for the 5090.
# - First gsplat import compiles kernels (slow once); subsequent runs are fast.
